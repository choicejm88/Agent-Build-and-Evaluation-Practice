# Stock News Analysis Skill

## 목적

사용자가 입력한 종목 또는 기업의 최신 뉴스를 기반으로 핵심 내용을 요약하고, 투자에 영향을 줄 수 있는 요소를 분석하여 투자 참고용 브리핑을 생성한다.

이 Skill은 투자 자문을 제공하지 않으며, 매수·매도·보유를 추천하지 않는다.

---

# 언제 사용하는가

다음과 같은 요청이 들어오면 이 Skill을 실행한다.

- 삼성전자 최근 뉴스 분석해줘.
- SK하이닉스 최신 뉴스 알려줘.
- 테슬라 최근 이슈를 정리해줘.
- AI 반도체 관련 주요 뉴스를 알려줘.
- 개장 전 뉴스 브리핑을 만들어줘.
- OO기업 주가에 영향을 줄 만한 뉴스가 있어?

---

# 입력(Input)

필수

- 종목명 또는 기업명

선택

- 티커(Symbol)
- 조회 기간
    (기본 : 최근 7일)
- 관심 분야
    (예: 실적, 정책, 공급망, 경쟁사)

---

# 사용할 Tool

## search_stock_news

MCP Tool

최신 뉴스를 조회할 때 반드시 호출한다.

Tool

search_stock_news(company)

Input

- company

Output

- 기사 제목
- 기사 링크
- 기사 날짜
- 기사 요약

이 반환값은 Observation으로 사용한다.

# Tool 사용 규칙

최신 뉴스가 필요한 질문에서는 반드시 search_stock_news Tool을 호출한다.

Tool이 반환한 Observation을 기반으로 분석한다.

Observation에 없는 내용을 추측하여 작성하지 않는다.

출처(URL)가 없는 뉴스는 사용하지 않는다.
---

## Tool 선택 규칙

① Naver OpenAPI 사용 가능

→ Naver OpenAPI 우선 사용

② Naver OpenAPI 미설정

→ Web Search Tool 사용

③ 둘 다 사용 가능한 경우

→ 결과를 비교하여 중복을 제거한 후 함께 활용한다.

# 수행 절차(Workflow)

## Step 1

사용자의 질문을 분석한다.

- 종목명 확인
- 기업명 확인
- 조회 기간 확인

필요 시 한 번만 추가 질문한다.

---

## Step2

Action

search_stock_news MCP Tool을 호출한다.

↓

Observation

반환된 뉴스 목록을 확인한다.

↓

Thought

뉴스를 정리하고 분석한다.
---

## Step 3

### Thought

Observation 결과를 검토한다.

### Observation 확인

Tool이 반환한 뉴스를 확인한다.

예)

```text
Observation

기사 수 : 8건

중복 기사 : 2건

유효 기사 : 6건
```

### Action

다음을 수행한다.

- 중복 기사 제거
- 동일 사건 통합
- 관련 없는 기사 제거
- 최신순 정렬

---

## Step 4

### Thought

정리된 뉴스를 기반으로 기업에 미치는 영향을 분석한다.

Observation에서 확인한 뉴스를 사용하여 아래 항목을 작성한다.

- 핵심 내용
- 관련성
- 영향 방향
- 영향도(1~5)
- 근거

추측하지 않는다.

Observation에 없는 사실은 추가하지 않는다.
---

## Step 5

최종 브리핑을 생성한다.

아래 템플릿을 따른다.

① 핵심 요약

② 주요 뉴스

③ 뉴스별 영향 분석

④ 투자 시 고려사항

⑤ 종합 의견

⑥ 출처

# 반드시 지켜야 하는 규칙

- 출처(URL)가 없는 정보는 사용하지 않는다.
- 원문에 없는 숫자나 사실을 생성하지 않는다.
- 확인되지 않은 정보는 **[확인 필요]** 로 표시한다.
- 뉴스와 의견을 명확히 구분한다.
- 매수·매도·보유를 추천하지 않는다.
- 주가 상승 또는 하락을 단정하지 않는다.
- 영향 방향에는 반드시 근거를 함께 작성한다.
- 같은 사건은 하나의 뉴스로 통합한다.
- 모든 결과는 한국어로 작성한다.
